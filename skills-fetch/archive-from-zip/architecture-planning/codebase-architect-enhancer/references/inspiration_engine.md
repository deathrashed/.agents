# The Inspiration Engine: Web & GitHub Sourcing

## Overview
Do not rely solely on pre-trained knowledge. Actively search the web and GitHub to find the most modern, efficient, and robust solutions for the codebase being audited. Your goal is to cross-pollinate the best industry practices directly into the user's repository.

## Execution Strategy

### 1. Identify Search Targets
- Extract the core domain and technologies of the current project.
- Identify the specific problems found during the audit (e.g., "React state management performance", "Python concurrent API requests", "anti-patterns in Express").

### 2. Execute Searches & Cross-Repo Importing
- Use web search tools to find "best practices", "modern architectures", and "open source alternatives" for the specific domain.
- Search GitHub for repositories solving the same overarching goal. Look for how top-tier open-source projects structure their code.
- Find industry-standard boilerplates, starter kits, or utility scripts that do exactly what the user is trying to do, but better.

### 3. Provide Links and Integration Instructions
- In your report, provide direct links to the specific GitHub repos, tools, or documentation that inspired your suggestions.
- Detail exactly how to import or integrate those workflows, utility scripts, or architectural patterns into the current codebase. Do not just drop a link; explain the integration path.

### 4. Adapt and Implement
- Do not blindly copy-paste. Adapt the discovered concepts to fit the exact context and constraints of the user's project.
- Translate external snippets into the project's language and style guidelines.
- Always include actionable code blocks so the user can immediately apply the newly discovered paradigm.
