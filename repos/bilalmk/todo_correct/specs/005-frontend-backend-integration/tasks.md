## ⚠️ Important: Better Auth UUID Integration Applied

**Status**: ✅ Completed (2026-01-02)

**Key Changes Made**:
1. Extended Better Auth `user` table with UUID column (auto-generated by database)
2. Updated all foreign keys to point to `user.uuid` instead of `users.id`
3. Dropped custom `users` table (Better Auth is now single source of truth)
4. Modified User model to map to Better Auth schema
5. Updated JWT validation to extract `uuid` custom claim
6. Removed backend auth endpoints (Better Auth handles registration/login)

**Reference**: See `BETTER_AUTH_UUID_INTEGRATION.md` and `data-model.md` for complete details

**For Future Task Execution**: All user-related code must use Better Auth `user` table with UUID column. See implementation guide for patterns.
